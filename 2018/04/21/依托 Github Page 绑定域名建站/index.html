依托 Github Page 绑定域名建站

发表于 2018-04-21 更新于 2019-12-08 分类于 建站 阅读次数：
本文字数： 1.3k 阅读时长 ≈ 1 分钟

事由

  • 这个博客刚开不久西部数码的空间就到期了，后来开通了 Google Cloud 试用一年，最近又到期（捂脸），遂打算转移 Github
  • 使用Github的好处：
    1. 开通 Github 个人主页超级简单，并且可以和自己的域名绑定
    2. Github 免费 1G 的代码仓库，用作个人博客空间绰绰有余
    3. 超便捷的更新部署，还能做版本控制
  • 下面开始动手！

    准备

  1. 首先你得有一个 Github 账号
  2. 其次你的电脑上需要安装 Git 命令行环境

部署

  1. 建立 Github 个人主页，成功后直接访问 “http://你的昵称.github.io”就可以，具体步骤按照下图标注：

  2. 进入刚刚新建的项目仓库，右上角选择settings，下滑到 GitHub Pages 标题处，不出意外的话已经有一行绿色的Your site is published at http://你的昵称.github.io，点击进去，看到该域名已经可以访问了。如果没有的话，在source下选择 master branch，然后点击 Save 即可。

  3. Git 环境配置 依次使用如下命令：

    1. 生成SSH密钥对

    2. 复制公钥

    3. 拷贝公钥

      • 在Github你的头像上点击，依次进入Settings – SSH and GPG keys – New SSH key，粘贴刚才拷贝的公钥，确认

    4. 连接 Github

      第一次会提示无法连接，这是正常现象，接着输入yes，回车，稍等片刻就好了

    5. 配置个人信息

  4. 现在本机环境已经能够通过SSH和Github通信了，下面配置Hexo的配置文件

    这里的昵称是Github昵称，不是刚才用命令配置的那个用户名，不放心的话可以在项目主页右上角复制过来。

  5. 安装部署插件

    接着直接hexo g -d就可以一键生成并部署到你的GitHub项目上了，并且GitHub主页的内容会根据项目文件实时更新

  6. 域名绑定

    • 使用自己的域名看起来更加高大上一点，并且也比“你的昵称.github.io”方便记忆，需要两个过程：
    1. 在..\博客文件夹\source\目录下新建一个名为CNAME的 文件，内容为你的个人域名：
    2. 域名解析，以西部数码为例，建立两条CNAME解析，主机名分别为@和www，对应值都是你的 Github 个人主页地址

  7. 最后hexo g -d，部署成功后浏览器访问你的域名就可以啦，是不是很爽呢！
