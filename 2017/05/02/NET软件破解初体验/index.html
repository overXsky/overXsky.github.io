.NET软件破解初体验

发表于 2017-05-02 更新于 2018-12-08 分类于 逆向 阅读次数：

事情源起

前几天晚上和朋友闲聊，聊到京东时不时会发一些神券，抢到就是赚到，无奈这种好事总是与我擦肩而过…

然后朋友就把我拉进了群，开始还有点小兴奋，拉了好几个人一起进来，结果待了两天发现里面发的都是些不痛不痒的东西，接着被我拉进来的朋友开始吐槽了…

很可惜，羊毛群收够人不再收了，在好奇心的驱使下，我本着爱踢行业人员自力更生的优良传统美德，动用搜索引擎大法开始查找羊毛线报相关的信息，找了一大圈最后还是回到了那个让我又爱又恨的赚吧 —— 这个自从我注册以来就一直没有各种权限的神奇论坛。。。

What is 赚吧？

赚吧，号称华人第一赚客论坛，吧内自古以来流通着一种名为“果果”的神奇物质。然而，自从入住赚吧以来我的果几乎从未涨过。原因无外乎做任务太慢，直接充钱太傻，购物返果却说查不到plus会员单号（黑人问号？？？）…说到底就是各种懒得弄。不过赚吧内的各种活动线报真心不是盖的，吧内大牛分分钟可以撸掉ofo红包车几万、p2p理财线下超市某东某宁某支付各种羊毛满天飞…（有没有人带带我(✪ω✪)）

赚吧怎么玩

炼气期玩家：RSS订阅

  • 作为一名新时代高科技分子，不动用一点自动化技术都不好意说自己是研究计算机的。最先想到的自动化技术就是RSS+正则+邮箱转发，很幸运，赚吧已经自带了 RSS 订阅，只需要用支持 RSS 的客户端订阅一下就可以了。

  • 这里我用的是 Inoreader ，一个 Web 端轻量级 RSS 客户端。我把要订阅的页面收藏到一个新建的名为 E-mail 文件夹内，如下图：

  • 接着点击右上角的小齿轮图标，依次进入偏好设置——规则——创建：新规则，按照下图配置新的订阅规则，正则表达式就输入想要关注的关键字，用|隔开，电子邮箱填写自己的常用邮箱，最后点击保存，页面自动返回到创建前的界面。

规则右边有四个图标，其中第一个按钮的功能是用来激活或者停用规则，一个小圈带个实心圆点就表明规则正常运行了。

  • 还可以进一步设置来减短 RSS 更新的周期。依次点击偏好设置——订阅源，找到之前订阅的赚客吧源，点击源的名称，在弹出来的详细信息窗口下方，增强周期为每10分钟更新一次（默认半小时）

  • 但是用了一天以后我就发现，虽然我的邮箱确实能够时不时地收到一些线报了，但是像一些抢券、红包等时效性强的信息经常都来不及领就没了，说到底还是 RSS 抓取的不及时导致的，我又不可能时时刻刻守在电脑前像个傻子不停的按 F5 吧…看来需要一种新的方式

筑基期玩家：VisualPing

  • 何为VisualPing？它在官网上的介绍是这样的：“We monitor webpages… so you don’t have to!”，翻译成人话：我们替你来监控网页。下面还有一行小字是：“Select an area and relax: We’ll send you an email alert when something changes”，对应人话：选择网页上一个区域，一旦有变化就可以发邮件告诉你。

  • 它通过浏览器插件的方式来运行。目前 Firefox 尚不支持，所以我用 Chrome 来测试了一下，安装完成以后右上角多了一个蓝色的波形图标，使用逻辑很简单：选择需要监控的网页区域——设置监控时间间隔——保存设置开始监控就能监控对应的网站了，属于一种比较通用型的解决方案。

  • 该方法确实要比 RSS 及时了很多，但是经过我的测试发现，有时候只是帖子多了几个回复而已它就像个2b一样使劲在右下角闪个不停，我们一般想要的都是对新发表的帖子提醒不是么？所以之后我也没去试云端功能，估计还要注册收费一大堆的。就不再废话了。

  • 磨叽了一大堆，下面有请本文的主角隆重登场

元婴期玩家：PostMonitor

  • 线报商品监控神器 PostMonitor 介绍：

PostMonitor：线报&商品监控神器，精雕细琢，史上最强！全网最牛！
支持赚客吧、线报屋、赚客之家、鲁大客、抢抢网、赚客网论坛帖子监控！
支持爱Q生活网、dir28活动资讯网、寻宝阁、券妈妈等线报监控！
支持京东、苏宁、国美、一号店等商品价格&库存监控(支持邮件提醒)！​
强大的关键词&黑名单自动筛选、新贴提醒（弹窗、声音、图标闪烁等）、快速预览、回贴、提醒抢果、闹钟提醒等功能！
软件支持锁定、支持快捷键操作、帖子收藏等人性化功能。

  • 更多的功能直接去上手体验就知道了，反正我用的是非常爽，可惜免费用户只有一个小时的高级VIP体验时间，到点就得忍受各种“不公正”待遇，很难受。

  • 一年的高级 VIP 更是高达 180 元每年，骇人听闻啊！下面就动手开始来破解~

破解思路

查壳&脱壳

  • 首先上 Detect It Easy 查壳：

  • 用的是 .NET Reactor 4.8-4.9 版本的保护，直接上 Reactor 专属脱壳神器 de4dot，理论上5.0以下的壳都能完美脱掉，试一下：

  • 脱完壳在同级目录下生成了一个加上 cleaned 后缀的文件，执行一下，能够成功运行则说明没问题

反编译&反汇编

  • 外壳脱完了，接下来就是逆向了，这里强烈推荐 Reflector ，虽然是收费软件，不过天朝国情你懂的嘛 一款反汇编的开源软件 dnSpy，项目主页在 github 上，它的调试功能非常强大，而且性能强劲，UI 让人赏心悦目
  • 这里我用的是 Reflector 8.5.0.179 版本，用于演示的是试用版本。其实只要不是太旧应该都没问题，正版试用版区别不大（有pj强迫症的同学请自行李彦宏）。
  • 下载好dnspy后得到一个包含大量文件的文件夹，最重要的两个文件是dnSpy.exe以及dnSpy-x86.exe前者用于逆向64位程序，后者用于逆向32位程序。因为我们这里软件是32位的，所以打开.\dnSpy\dnSpy-x86.exe，载入我们脱壳后的文件，项目目录结构、文件名称、反编译源码都一目了然地呈现在面前.

寻找破解突破点&修改 IL 指令

  • 首先我们以免费用户身份运行一下软件，寻找一下破解的突破口。登录软件以后右下角提示说，“您的VIP授权已到期…”

  • 使用搜索功能，查找字符串“VIP授权”，一共有7个位置出现了这个关键字，我们依次双击查看，发现有一个变量 isExpiry 出现多次。在最后一个函数中，它作为 if 语句的判断条件，当它为真时就会出现对用户 VIP 授权到期相关的提醒。并且从字面意思上也可以知道，它是用来存储用户 VIP 是否过期的一个布尔型变量。

  • 找到了关键入口点，我们在 isExpiry 上右键–分析，或者使用快捷键Ctrl + Shift + R
    ，来观察这个变量被读取和赋值的情况。这个功能是 dnSpy 特有的，十分强大。从业务逻辑上分析不难得出，对于这个变量我们最关注的是对它进行赋值的过程，因为这直接关系到了当前用户的用户组权限问题。所以我们重点开始关注赋值部分，一共有4个函数对它进行了赋值。

  • 我们就以第一个函数的第一处语句为例，来说明一下对于IL语句的修改方式。

    • 这几条语句的意思很明显，首先从服务器上获取用户 VIP 的到期时间，存储在本地 minValue 变量当中，之后将 minValue 与当前系统时间进行对比，那么可想而知得到的结果是 True。

缓解措施

  • 这一小节我们针对上述内容讲一讲，作为一个 .NET 软件开发者如何防范自己的软件被破解
    1. 加壳，加强壳！.NET Reactor 什么的太弱了好吗！！！这种有脱壳工具的加固方式就不要再用了，VMP 就是一个很不错的选择。
    2. 代码混淆！！！自己在做开发的时候变量名通俗易懂是好事，但是发布出去前，请拷贝一个副本然后把代码混淆一下！！！不然别人一眼就知道这个函数/变量/方法的作用了好吗！！！当然花指令什么的我觉得用处不大，随意就好。
    3. 加强验证！！！只把验证放在本地的都是小白行为，服务器上加强验证手段才是王道。对于本例，免费用户只具有每两分钟请求一次“赚客大家谈”更新的权限，如果一个免费用户突破了两分钟的限制，或者获取了其他越权信息，那么就可以判断它为非法用户。相应的，我们修改客户端请求消息的格式，每条消息需要带上它的用户名，服务器收到后，首先判断该用户所处的用户组，如果非 VIP 用户请求了 VIP 才能获得的信息，那么就不予以响应，甚至把该用户拉黑。

后话

  • 破解基于 .NET 框架的软件相对还是较为容易的，我们不用费太大功夫就能阅读到源码，可以基于源码直接进行修改。
  • 想要熟练地逆向 .NET，首先要熟悉 IL 语言中的常用语句（赋值、条件语句、循环语句、判断、数据类型等等），这里给一篇参考：IL指令详细。
  • 其次要熟练使用逆向工具，dnSpy 就是一个很好的学习对象，在本文中只用到了最基础的搜索、分析、反编译、修改IL指令功能，它强大的跟踪调试功能才是最为亮眼的利器。另外，运气好的情况下我们可以直接修改反编译出来的源代码然后编译，而不必去修改相对复杂的 IL 指令。但是很多情况下都会因为反编译的不完善而编译报错，最简单的测试办法就是反编译完一个函数直接右键–修改方法，然后什么也不改动直接 compile，如果没报错那么恭喜你这个函数可以直接修改源码了！但更多的情况都是不行的，所以大家老老实实学习IL指令才是正解。
  • 逆向一道没有捷径，需要多动手，学习 IL 和 dnspy 的最好方式是多动手去调，遇到不会的去查，不要怕出错，而不是先找一堆教程干瞪眼去看，这样学习逆向是毫无用处的。
